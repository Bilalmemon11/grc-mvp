### âŒ DON'T Commit:
1. `node_modules/` - Always ignored
2. `.env` files - Contains secrets
3. Build outputs (`dist/`, `build/`) - Generated code
4. IDE settings - Personal preferences
5. `info/` folder - Internal planning

### âœ… DO Commit:
1. `package-lock.json` - Ensures reproducible builds
2. `prisma/migrations/` - Database version control

4. All source code

## Azure Deployment Considerations

For Azure deployment, these files are essential:

### âœ… Must Commit:
- `backend/package.json` + `package-lock.json`
- `frontend/package.json` + `package-lock.json`
- `backend/prisma/schema.prisma`
- `backend/prisma/migrations/`
- All source code in `src/` directories
- Docker configurations (if using containers)

### âš™ï¸ Set in Azure:
- Environment variables (`.env` values)
- Database connection strings
- Azure OpenAI credentials
- Storage connection strings

### ğŸ“‹ Deployment Process:
1. Push code to Git (GitHub/Azure Repos)
2. Azure pulls code
3. Runs `npm install` (uses `package-lock.json`)
4. Runs `npx prisma migrate deploy` (uses migrations)
5. Runs `npm run build` (creates `dist/`)
6. Starts application

## Three-Level .gitignore Structure

This project uses a hierarchical approach:

### 1. Root `.gitignore`
- Global ignores for entire project
- Dependencies, build outputs, env files
- Project planning (`info/`)

### 2. `backend/.gitignore`
- Backend-specific ignores
- Prisma cache, local databases
- Backend temp files

### 3. `frontend/.gitignore`
- Frontend-specific ignores
- Vite cache, build stats
- Frontend temp files

**Priority**: More specific (backend/frontend) overrides root
